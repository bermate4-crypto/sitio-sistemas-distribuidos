1. ¿Qué ventajas encuentras en trabajar directamente en la nube con GitHub?
Trabajar en la nube con GitHub permite acceder al proyecto desde cualquier lugar sin necesidad de configurar un entorno local.
Facilita la colaboración en tiempo real, el control de versiones y la integración continua. Además, herramientas como GitHub Pages permiten desplegar sitios web rápidamente sin infraestructura adicional.
2. ¿Qué desafíos tuviste al crear el sitio sin usar herramientas locales?
 Uno de los principales desafíos fue la limitación en la personalización del entorno de desarrollo, ya que no se tiene el mismo control que en una máquina local.
También puede haber dificultades para depurar errores, especialmente si no se cuenta con herramientas avanzadas de testing o servidores locales.
La dependencia de la conexión a internet también puede afectar la productividad.
3. ¿Cómo podrías mejorar este sitio web en el futuro?
Se podría mejorar el diseño haciéndolo más atractivo y responsivo, optimizar el rendimiento (tiempos de carga), e incorporar funcionalidades dinámicas como formularios o integración con APIs.
También sería útil mejorar la accesibilidad y el SEO para llegar a más usuarios. Implementar buenas prácticas de desarrollo y pruebas automatizadas también ayudaría a mantener la calidad del sitio.
4. ¿Qué aprendiste sobre Sistemas Distribuidos al crear este sitio?
Aprendí que plataformas como GitHub funcionan como sistemas distribuidos, donde múltiples servidores gestionan repositorios,
versiones y despliegues. Entendí conceptos como la replicación de datos, la tolerancia a fallos y la colaboración concurrente.
También vi cómo el control de versiones es clave para coordinar cambios entre múltiples usuarios en un entorno distribuido.
